## ⚙️ Setup & Installation

### 1. Clone / Navigate to the project
```bash
cd skillwallet
```

### 2. Create a virtual environment (recommended)
```bash
python -m venv venv
venv\Scripts\activate          # Windows
# source venv/bin/activate     # macOS/Linux
```

### 3. Install dependencies
```bash
pip install -r requirements.txt
```

### 4. Add the Kaggle datasets
Download from Kaggle: [Credit Card Approval Prediction Dataset](https://www.kaggle.com/datasets/rikdifos/credit-card-approval-prediction)

Place in `data/`:
```
data/application_record.csv
data/credit_record.csv
```

> **Note:** If CSVs are absent, `train.py` generates realistic **synthetic data** automatically so the full pipeline still runs for demonstration.

## 🤖 ML Pipeline

### Step 1 — Train the Model
```bash
python train.py
```

This runs the full pipeline:
1. Loads `application_record.csv` + `credit_record.csv` (or generates synthetic data)
2. Performs EDA — saves plots to `static/assets/eda/`
3. Executes `data_cleaning()` — deduplication, missing values, feature engineering
4. Creates binary target label using `is_risky()` (STATUS codes 2–5 = risky)
5. Merges datasets on `ID`
6. Label-encodes categorical columns
7. Trains 4 models: Logistic Regression, Decision Tree, Random Forest, XGBoost
8. Compares all by F1-score, tunes the best with `RandomizedSearchCV`
9. Saves `model.pkl` + `encoders.pkl`

### Step 2 — Launch the Flask App
```bash
python app.py
```

Open: **[http://127.0.0.1:5000](http://127.0.0.1:5000)**

## 📋 Input Features

### Personal
- Gender (M/F)
- Owns Car (Y/N)
- Owns Real Estate (Y/N)
- Mobile / Work Phone / Email flags

### Financial
- Total Annual Income
- Income Type (Working, Commercial associate, Pensioner, State servant, Student)
- Education Level (Higher, Secondary, etc.)
- Days Employed

### Family
- Marital Status
- Housing Type
- Age (in days)
- Number of Children
- Total Family Members

### Credit History
- EMIs Paid Off (months)
- EMI Overdues (months)
- Total Loan Accounts
- Earliest Credit Month
- Latest Credit Month

## 🎯 Target Label

STATUS code interpretation:
| Code | Meaning | Risk |
|---|---|---|
| C | Paid off that month | ✅ Good |
| X | No loan that month | ✅ Neutral |
| 0 | 1–29 days overdue | ⚠️ Low |
| 1 | 30–59 days overdue | ⚠️ Medium |
| 2 | 60–89 days overdue | ❌ **HIGH** |
| 3 | 90–119 days overdue | ❌ **HIGH** |
| 4 | 120–149 days overdue | ❌ **HIGH** |
| 5 | Bad debt / write-off | ❌ **HIGH** |

**Binary target:** `1` = ever had a status of 2/3/4/5 → Rejected; `0` = otherwise → Approved

## 🗃️ Database / ER Design

5 entities:

| Entity | PK | Key Attributes |
|---|---|---|
| Users | UserID | Name, Email, Password, Role |
| Applicant_Details | ApplicantID (FK: UserID) | IncomeType, EducationType, EmploymentDays |
| Credit_History | HistoryID (FK: ApplicantID) | MonthBalance, PaymentStatus, OverdueStatus |
| ML_Model | ModelID | ModelName, AlgorithmType, Accuracy |
| Approval_Prediction | PredictionID (FK: ApplicantID, ModelID) | ApprovalResult, RiskCategory, PredictionDate |
